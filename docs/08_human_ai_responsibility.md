# Human-AI Responsibility Split

The submission should show that AI assisted the team while humans kept control over product direction, scope, data interpretation, and risk decisions.

## Delegated to AI

- Brainstormed product names and feature options.
- Compared PRD alternatives.
- Generated UI scaffolding and documentation drafts.
- Suggested architecture trade-offs.
- Drafted user-facing explanations and roadmap language.
- Helped implement tests, backend endpoints, resume parsing, GitHub profile scanning, Exa integration, and Apify normalization.

## Human-judged

- Final product direction.
- Track selection.
- Scope control.
- Ethical guardrails.
- Dataset interpretation.
- Recommendation quality.
- Whether outputs were safe, useful, and explainable.

## Responsible AI Principle

Generated text may explain deterministic recommendations. It should not decide scores, invent evidence, or claim guaranteed career outcomes.

## Current Safeguards

- SkillsFuture data drives role definitions and target skill gaps.
- Resume/manual/GitHub evidence only affects scoring when matched to target-role vocabulary.
- Exa and Apify enrich the analysis but do not replace dataset scoring.
- Saved roadmaps preserve the user's chosen path and evidence snapshot.
